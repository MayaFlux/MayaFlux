22/**
23 * @class Engine
24 * @brief Central lifecycle manager and component orchestrator for the MayaFlux processing system
25 *
26 * The Engine serves as the primary entry point and lifecycle coordinator for MayaFlux, acting as:
27 * - **Lifecycle Manager**: Controls initialization, startup, pause/resume, and shutdown sequences
28 * - **Component Initializer**: Creates and configures core system components with proper dependencies
29 * - **Access Router**: Provides centralized access to all major subsystems and managers
30 * - **Reference Holder**: Maintains shared ownership of core components to ensure proper lifetime management
31 *
32 * **Core Responsibilities:**
33 * 1. **System Initialization**: Orchestrates the creation and configuration of all core components
34 * 2. **Lifecycle Control**: Manages the start/stop/pause/resume cycle of the entire processing system
35 * 3. **Component Access**: Provides unified access to subsystems (audio, scheduling, node graph, buffers)
36 * 4. **Resource Management**: Ensures proper construction/destruction order and shared ownership
37 *
38 * **Architecture Philosophy:**
39 * The Engine follows a "batteries included but replaceable" approach:
40 * - Provides sensible defaults and automatic component wiring for ease of use
41 * - Allows advanced users to access individual components directly for custom workflows
42 * - Enables completely custom component instantiation when needed
43 *
44 * **Usage Patterns:**
45 *
46 * *Simple Usage (Recommended):*
47 * ```cpp
48 * Engine engine;
49 * engine.Init(48000, 512, 2, 0); // 48kHz, 512 samples, stereo out
50 * engine.Start();
51 * // Use engine.get_scheduler(), engine.get_node_graph_manager(), etc.
52 * ```
53 *
54 * *Advanced Usage:*
55 * ```cpp
56 * Engine engine;
57 * auto custom_scheduler = std::make_shared<CustomScheduler>();
58 * engine.Init(stream_info);
59 * // Replace default scheduler with custom implementation
60 * engine.get_scheduler() = custom_scheduler;
61 * ```
62 *
63 * *Offline Processing:*
64 * ```cpp
65 * // Engine components can be used without hardware I/O
66 * auto scheduler = engine.get_scheduler();
67 * auto node_graph = engine.get_node_graph_manager();
68 * // Process manually without Start()
69 * ```
70 *
71 * The Engine does not perform direct signal processing or scheduling - it delegates these
72 * responsibilities to specialized subsystems while ensuring they work together coherently.
73 */
