28/**
29 * @class TextureLoom
30 * @brief Portal-level texture creation and management
31 *
32 * TextureLoom is the primary Portal::Graphics class for creating and
33 * managing GPU textures. It bridges between user-friendly Portal API and
34 * backend VKImage resources via BufferService registry.
35 *
36 * Key Responsibilities:
37 * - Create textures (2D, 3D, cubemaps, render targets)
38 * - Load textures from files (delegates to IO namespace)
39 * - Manage sampler objects (filtering, addressing)
40 * - Track texture lifecycle for cleanup
41 * - Provide convenient format conversions
42 *
43 * Design Philosophy:
44 * - Manages creation, NOT rendering (that's Pipeline/RenderPass)
45 * - Returns VKImage directly (no wrapping yet)
46 * - Simple, focused API (file loading deferred to IO)
47 * - Integrates with BufferService for backend independence
48 *
49 * Usage:
50 * auto& mgr = Portal::Graphics::TextureLoom::instance();
51 *
52 * // Create basic texture
53 * auto texture = mgr.create_2d(512, 512, ImageFormat::RGBA8);
54 *
55 * // Create with data
56 * std::vector<uint8_t> pixels = {...};
57 * auto texture = mgr.create_2d(512, 512, ImageFormat::RGBA8, pixels.data());
58 *
59 * // Create render target
60 * auto target = mgr.create_render_target(1920, 1080);
61 *
62 * // Get sampler
63 * SamplerConfig config;
64 * config.mag_filter = FilterMode::LINEAR;
65 * auto sampler = mgr.get_or_create_sampler(config);
66 */
