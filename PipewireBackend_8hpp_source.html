102/**
103 * @class PipewireStream
104 * @brief PipeWire implementation of the audio stream interface
105 *
106 * Wraps a pw_stream follower node. The stream proposes SPA_AUDIO_FORMAT_F64
107 * interleaved during format negotiation, matching the engine's internal
108 * double representation and eliminating format conversion in the hot path.
109 *
110 * On param_changed the negotiated frame count is stored and used to validate
111 * the buffer size contract. If PipeWire adjusts the quantum away from the
112 * requested value, a warning is logged and m_stream_info.buffer_size is
113 * updated to match, consistent with the RtAudio backend's existing behaviour.
114 *
115 * The pw_thread_loop drives the PipeWire event loop on a dedicated thread.
116 * The process callback runs on that thread at SCHED_FIFO priority when
117 * rlimits are configured by the package installer. No locking occurs on
118 * the audio path; the process callback only reads atomic state and invokes
119 * the registered std::function.
