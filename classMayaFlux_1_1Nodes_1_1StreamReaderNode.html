Detailed Description

Node that reads sample-by-sample from externally provided data.

Exposes set_data() to receive a block of samples from any source. Each process_sample() advances a read head through that data. When exhausted, returns the last valid sample (hold).

The node does not know or care who calls set_data(). A buffer processor, a pipeline dispatch, a coroutine, manual user code, anything that has a span of doubles can feed it.

Ownership: only one writer may feed data at a time. The first caller to set_data() with a non-null owner claims exclusive write access. Subsequent callers with a different owner are rejected until release_owner() is called.
