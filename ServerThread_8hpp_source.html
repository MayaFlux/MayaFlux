85 * Implementation Notes:

87 * - Uses shared_ptr<atomic<bool>> to ensure stop flag outlives thread
88 * - Memory ordering: acquire on read, release on write (matches std::stop_token semantics)
89 * - Minimal overhead: one atomic load per loop iteration (~1-2 CPU cycles)
